StaffPunch

Development Period: 2013-2014 Status: Inactive

Project Scope

StaffPunch came to be out of internal necessity and a desire to want something a little different that what was available. during my tenure with SDF Technical we made a challenge for ourselves in between projects. The ultimate end goal was to make a product. Something that we could not only use, but sell and offer to other businesses. We figured that a year would have been a sufficient timeline set in order to launch.

We began with writing down what we needed the application to accomplish:

  • A running clock
  • a method to "punch in" and "punch out"
  • break and lunch handling
  • user roles and administration
  • department/groups administration
  • reporting

Research

We scoured the internet to see how other applications worked out time tracking. We found that Toggl, TimeDoctor and Harvest had the all of the features that we were looking to emulate. Going by what we had found we wanted to make something simpler looking that had room to grow.

Prototyping

I, then, became responsible for taking the information that we had gathered and coming up with a core UI schema. The idea of a visual representation of time was to be the main focus. Instead of a simple ticking clock, the goal was to show the user where they were in the day; designated by a red line, and grow their active statuses on the timeline as the day progressed.

Making different UI options for application user roles became apparent. What would a standard user see? What would an administrative user see. What about a user that had a more flexible role like a moderator or some other middle-management?

Functional Development

Timing the line animation to only move as a clock was quite possiply the most difficult feature. There were initial settings that had to be taken into account such as user time zone in order to line up everything properly. After that it was a matter of using a javascript plugin called Moment to connect that to the front end UI display.

Once the milestones of allowing the user to log in to the application, "Punch In", "Punch Out" and "Go to Lunch" were completed it came down to more UI refinements and adding in the task activities.

Wrap Up

The front end of the site was the last thing to be fully assembled. While the rest of the application was being constructed, the marketing copy was still being discussed and decided. Logo design went through many revisions until we finally came together on design that we liked.
